# Skills de UI/UX

Conjunto de skills que guiam o desenvolvimento de componentes bonitos, consistentes e acessíveis neste repositório (Next.js 15 + Payload CMS 3 + Tailwind v4).

Cada skill é um `SKILL.md` independente, com frontmatter YAML e um checklist objetivo. Use-as nesta ordem ao construir/revisar UI:

| # | Skill | Quando usar |
|---|-------|-------------|
| 1 | [🎨 design-system-architect](./design-system-architect/SKILL.md) | Ao definir ou estender tokens (cor, tipografia, espaçamento, radius, elevação). |
| 2 | [⚛️ react-component-builder](./react-component-builder/SKILL.md) | Ao criar/refatorar um componente React. |
| 3 | [🔍 ui-consistency-reviewer](./ui-consistency-reviewer/SKILL.md) | Ao revisar visualmente uma tela/PR antes do merge. |
| 4 | [♿ accessibility-checker](./accessibility-checker/SKILL.md) | Ao auditar a acessibilidade (WCAG 2.2 AA). |

## Fontes de referência

- WCAG 2.2 (W3C) — <https://www.w3.org/WAI/WCAG22/quickref/>
- Material Design 3 — <https://m3.material.io/foundations>
- Refactoring UI (Adam Wathan & Steve Schoger) — princípios de hierarquia, cor e espaçamento.
- Design Systems by Figma — <https://www.designsystems.com/>
- Inclusive Components (Heydon Pickering) — <https://inclusive-components.design/>
- ARIA Authoring Practices Guide (APG) — <https://www.w3.org/WAI/ARIA/apg/>
- Radix UI Primitives — <https://www.radix-ui.com/primitives>
- Tailwind CSS v4 docs — <https://tailwindcss.com/docs>
